attach(...)

 

Attaches to the process specified in pid, making it a traced "child" of the calling process; the behaviour of the child is as if it had done a pinktrace.trace.me(). The child is sent a SIGSTOP, but will not necessarily have stopped by the completion of this call; use wait(2) to wait for the child to stop.

Parameters:
  • pid - Process ID of the traced child
Raises:
  • OSError - Raised when the underlying ptrace(2) call fails.

cont(...)

 

Restarts the stopped child process.

Parameters:
  • pid - Process ID of the child to be restarted
  • sig - If this is non-zero and not SIGSTOP, it is interpreted as the signal to be delivered to the child; otherwise, no signal is delivered. Thus, for example, the parent can control whether a signal sent to the child is delivered or not. (Optional, defaults to 0)
  • addr - On FreeBSD this argument is an address specifying the place where execution is to be resumed (a new value for program counter), or 1 to indicate that execution is to pick up where it left off. On Linux, this argument is not used. (Optional, defaults to 1)
Raises:
  • OSError - Raised when the underlying ptrace(2) call fails.

detach(...)

 

Restarts the stopped child as for pinktrace.trace.cont(), but first detaches from the process, undoing the reparenting effect of pinktrace.trace.attach().

Parameters:
  • pid - Process ID of the traced child
  • sig - Treated the same as the signal argument of pinktrace.trace.cont()
Raises:
  • OSError - Raised when the underlying ptrace(2) call fails.

See Also: pinktrace.trace.cont

geteventmsg(...)

 

Retrieve a message (as an unsigned long) about the trace event that just happened, placing it in the location given by the second argument. For EXIT event this is the child's exit status. For FORK, VFORK, CLONE and VFORK_DONE events this is the process ID of the new process.

Parameters:
  • pid - Process ID of the traced child
Returns: long
The event message
Raises:
  • OSError - Raised when the underlying ptrace(2) call fails.

Note: Availability: Linux

me(...)

 

Indicates that this process is to be traced by its parent. Any signal (except SIGKILL) delivered to this process will cause it to stop and its parent to be notified via wait(2). Also, all subsequent calls to execve(2) by this process will cause a SIGTRAP to be sent to it, giving the parent a chance to gain control before the new program begins execution.

Raises:
  • OSError - Raised when the underlying ptrace(2) call fails.

resume(...)

 

Resumes the stopped child process. This is equivalent to pinktrace.trace.cont(pid, sig, 1).

Parameters:
  • pid - Process ID of the child to be restarted
  • sig - If this is non-zero and not SIGSTOP, it is interpreted as the signal to be delivered to the child; otherwise, no signal is delivered. Thus, for example, the parent can control whether a signal sent to the child is delivered or not. (Optional, defaults to 0)
Raises:
  • OSError - Raised when the underlying ptrace(2) call fails.
